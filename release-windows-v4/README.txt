本版本修复了 React Error #185 的根本原因：

技术细节
================================================================================

React Error #185 = "Maximum update depth exceeded"（最大更新深度超出）

根本原因：
1. MainWorkspaceView 中 connections 和 displayAgents 每次渲染都重新计算
2. 这些新数组作为 props 传递给子组件
3. 子组件接收到新的 props 引用，触发重新渲染
4. 重新渲染导致父组件也重新渲染
5. 形成无限循环

修复方案：
- 使用 useMemo 缓存 connections 计算结果
- 使用 useMemo 缓存 displayAgents 过滤结果
- 只有当依赖真正变化时才重新计算

使用方法
================================================================================

【方式一：便携版（推荐）】

1. 直接双击运行 AgentSwarmWritingSystem.exe
2. 首次运行会自动创建配置文件
3. 无需安装，可以放在任意目录运行
4. 确保 prompts 文件夹与可执行文件在同一目录

【方式二：安装版】

1. 双击运行 Agent Swarm Writing System_0.1.0_x64-setup.exe
2. 按照安装向导完成安装
3. 从开始菜单或桌面快捷方式启动程序

系统要求
================================================================================

- 操作系统：Windows 10/11 (64位)
- 内存：建议 4GB 以上
- 磁盘空间：至少 200MB 可用空间

首次使用配置
================================================================================

1. 启动程序后，点击右上角配置按钮
2. 配置 AI 服务：
   - 选择服务类型（OpenAI 或兼容服务）
   - 输入 API Key
   - 输入 Base URL（如果使用自定义服务）
   - 点击"测试连接"验证配置
3. 返回主界面，填写论文主题
4. 点击"开始写作"启动智能体协作

故障排除
================================================================================

【问题：程序无法启动】
- 确保 Windows 10/11 64位系统
- 检查是否被杀毒软件拦截
- 尝试以管理员身份运行

【问题：找不到 prompts 文件夹】
- 确保 prompts 文件夹与 .exe 文件在同一目录
- 不要移动或删除 prompts 文件夹

【问题：AI 服务连接失败】
- 检查网络连接
- 验证 API Key 是否正确
- 确认 Base URL 格式正确（需要包含 http:// 或 https://）

【问题：点击"开始写作"后程序卡住】
- 本版本已修复无限循环问题
- 如仍有问题，请检查 AI 服务配置是否正确
- 尝试重启程序
